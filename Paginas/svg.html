SVG (gráficos vetoriais escaláveis)

SVG define gráficos baseados em vetores em XML , que podem ser incorporados diretamente em páginas HTML.



O elemento <svg>

O elemento HTML <svg>é um contêiner para gráficos SVG. SVG possui vários métodos para desenhar caminhos, retângulos, círculos, polígonos, texto e muito mais.

Círculo SVG:

Exemplo:

<svg width="100" height="100">
<circle cx="50" cy="50" r="40" stroke="green" stroke-width="4" fill="yellow" />
</svg>



Retângulo SVG:

Exemplo:

<svg width="400" height="120">
<rect x="10" y="10" width="200" height="100" stroke="red" stroke-width="6" fill="blue" />
</svg>



Retângulo SVG com opacidade e cantos arredondados:

Exemplo:

<svg width="400" height="180">
<rect x="50" y="20" rx="20" ry="20" width="150" height="150"
style="fill:red;stroke:black;stroke-width:5;opacity:0.5" />
</svg>



Estrela SVG:

Exemplo:

<svg width="300" height="200">
<polygon points="100,10 40,198 190,78 10,78 160,198"
style="fill:lime;stroke:purple;stroke-width:5;fill-rule:evenodd;" />
</svg>



Elipse e texto gradiente SVG:

Exemplo:

<svg height="130" width="500">
<defs>
<linearGradient id="grad1">
<stop offset="0%" stop-color="yellow" />
<stop offset="100%" stop-color="red" />
</linearGradient>
</defs>
<ellipse cx="100" cy="70" rx="85" ry="55" fill="url(#grad1)" />
<text fill="#ffffff" font-size="45" font-family="Verdana" x="50" y="86">SVG</text>
Sorry, your browser does not support inline SVG.
</svg>

Diferenças entre SVG e Canvas

SVG é uma linguagem para descrever gráficos 2D em XML, enquanto o Canvas desenha gráficos 2D dinamicamente (com JavaScript).

SVG é baseado em XML, o que significa que todos os elementos estão disponíveis no SVG DOM. Você pode anexar manipuladores de eventos JavaScript a gráficos SVG.

No SVG, cada forma desenhada é lembrada como um objeto. Se os atributos de um objeto SVG forem alterados, o navegador poderá renderizar novamente a forma automaticamente.

A tela é renderizada pixel por pixel. No canvas, uma vez desenhado o gráfico, ele é esquecido pelo navegador. Se a sua posição for alterada, toda a cena precisa ser redesenhada, incluindo quaisquer objetos que possam ter sido cobertos pelo gráfico.
